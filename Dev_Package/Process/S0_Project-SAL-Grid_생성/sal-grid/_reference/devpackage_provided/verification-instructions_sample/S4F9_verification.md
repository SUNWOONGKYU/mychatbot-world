# S4F9: 사이드바 Project UI 개선 - 검증 지침

## 검증 정보
- **Task ID**: S4F9
- **Verification Agent**: code-reviewer

## 검증 항목

### 1. UI 배치 검증
- [ ] "+ 새로운 Project 등록" 버튼이 진행중 프로젝트 항목 **위**에 위치하는가?
- [ ] 버튼이 명확한 **버튼 UI** (배경색, 테두리, 호버 효과)를 가지는가?
- [ ] 기존 메뉴 항목과 시각적으로 구분되는가?

### 2. 액션 버튼 검증
- [ ] 진행중 프로젝트 항목 클릭 시 **두 개의 액션 버튼**이 표시되는가?
- [ ] "완료 처리" 버튼이 정상 작동하는가? (completeProject 호출)
- [ ] "폴더 연결" 버튼이 존재하는가? (기능은 S4F10에서 구현)
- [ ] 액션 버튼 토글(표시/숨김)이 정상 작동하는가?

### 3. 기존 기능 회귀 검증
- [ ] 프로젝트 등록 기능(registerNewProject)이 정상 작동하는가?
- [ ] 프로젝트 완료 처리 기능이 정상 작동하는가?
- [ ] 예시/완료 프로젝트 목록 토글이 정상인가?
- [ ] 동적 프로젝트 로딩(loadCurrentProject)이 정상인가?

### 4. 반응형 검증
- [ ] 모바일/태블릿에서 버튼이 깨지지 않는가?
- [ ] 사이드바 너비(280px) 내에서 레이아웃이 정상인가?

## 통과 기준

- 위 검증 항목 모두 통과
- 기존 기능(프로젝트 등록, 완료 처리, 목록 토글) 정상 작동
- 콘솔 에러 없음
