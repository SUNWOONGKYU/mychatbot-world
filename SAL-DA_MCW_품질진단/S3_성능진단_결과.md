# SAL-DA MCW — S3 성능 진단 결과

> Stage 3: 성능 진단 — 6카드
> 진단일: 2026-04-09

---

## PF (성능) — 6카드

### S3PF1: 번들 사이즈 최적화 — Warning / High
- Issues: dynamic import 1곳뿐 (create/page.tsx). 1500줄 chat-window 등 정적 import
- 21개 page.tsx가 'use client' → 서버 컴포넌트 혜택 없음
- CSS 13개 파일(10,752줄) layout.tsx에서 일괄 로드
- Impact: 초기 로딩 번들에 전체 코드 포함 → FCP 지연, 저사양 기기에서 페이지 렌더 느림
- Improvement Plan: chat-window, BotsTab 등 heavy 컴포넌트 dynamic import 적용, 'use client' 최소화, CSS 파일 route별 분할 로드

### S3PF2: 이미지/폰트 최적화 — Warning / High
- Issues: next/image 사용 2곳뿐. img 태그 width/height 미지정 → CLS
- Pretendard CDN @import → 렌더 블로킹 200~400ms, FOUT
- next/font 미사용. remotePatterns `hostname: '**'` 전체 허용
- Impact: CLS 점수 저하로 Core Web Vitals 불합격, 폰트 로딩 지연으로 텍스트 깜빡임, 보안 위험 외부 이미지 허용
- Improvement Plan: img → next/image 전환 (width/height 명시), Pretendard를 next/font/local로 교체, remotePatterns 도메인 화이트리스트 적용

### S3PF3: API 응답 시간 — **Danger** / **Critical**
- Issues: OpenRouter fetch 3곳 timeout/AbortController 없음 → hanging 위험
- vercel.json maxDuration 미설정
- .select() 189개 중 .limit() 체인 없는 uncapped 쿼리 다수
- Impact: LLM 지연 시 서버리스 함수 최대 실행 시간(60초) 소모 → 사용자 무응답 대기, 비용 낭비
- Improvement Plan: AbortController 30초 timeout 적용, vercel.json maxDuration 설정, uncapped 쿼리에 .limit() 추가

### S3PF4: 클라이언트 렌더링 효율 — Warning / Medium
- Issues: useCallback 153개 / useMemo 5개 / React.memo **0개**
- 1500줄 chat-window 전체 리렌더 (메시지 추가마다)
- inline onClick 핸들러 다수 (chat 7, BotsTab 11, jobs 9)
- Impact: 채팅 메시지 추가마다 1500줄 컴포넌트 전체 리렌더 → UI 버벅임, 특히 저사양 기기에서 체감 저하
- Improvement Plan: 메시지 리스트 컴포넌트 React.memo 적용, useMemo 확대, inline 핸들러 useCallback 전환

### S3PF5: 캐싱 전략 — **Danger** / High
- Issues: revalidate 설정 1곳뿐. SWR/React Query 미사용
- API Cache-Control 헤더 2곳만 (stream, tts). 나머지 전부 no-cache
- 매 방문 시 전체 re-fetch → Supabase 요청 폭증
- Impact: 동일 사용자 재방문마다 전체 데이터 재조회 → Supabase 요청 수 과금 폭증, 체감 로딩 속도 저하
- Improvement Plan: 정적 데이터 route에 revalidate 설정, 클라이언트 데이터 페칭에 SWR 도입, Cache-Control 헤더 전체 적용

### S3PF6: Core Web Vitals 추정 — Warning / High
- Issues: LCP: Pretendard FOUT + 이미지 lazy 미적용
- CLS: img width/height 미지정, 폰트 시프트
- INP: chat-window sendMessage 동기 복합 체인
- Impact: Google 검색 순위 하락 (CWV 신호 반영), 사용자 이탈률 증가 (LCP 2.5초 초과 시)
- Improvement Plan: Pretendard next/font 전환(LCP 개선), img width/height 일괄 지정(CLS 해소), sendMessage 비동기 분리(INP 개선)
