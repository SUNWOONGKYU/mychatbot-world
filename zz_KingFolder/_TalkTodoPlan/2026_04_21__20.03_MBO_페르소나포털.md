## 최상위 목표
"한 사용자의 모든 코코봇 페르소나(최대 10개)를 **한 화면의 탭 UI**로 노출하고, 탭 전환 시 대화 상태가 유지되는 **페르소나 포털** 페이지를 구축한다."

| 항목 | 현재 (AS-IS) | 목표 (TO-BE) |
|------|-------------|-------------|

## 기본 결정 (사용자 편의성 기준)

- **Q1 라우트**: 전용 `/hub` — 북마크·딥링크·GNB 진입점 부여 가능 (마이페이지 탭 깊숙이 묻히지 않음)
- **Q2 탭 상한**: 10개 하드리밋 + 초과 시 `...▼` 드롭다운 — 인지 과부하 방지
- **Q3 마운트 전략**: 활성 탭만 마운트(state 는 Map 으로 Ref 보존) — 메모리 절감 + 0ms 데이터 손실
- **Q4 모바일**: 가로 스크롤 sticky 탭바 (iOS Safari native feel) + 44px 터치 타겟 — 한 손 조작 최적화

## 리스크

| 리스크 | 대응 방안 |
|--------|----------|

| 10개 초과 시 UX 혼란 | 오버플로우 드롭다운 + 마지막 활동 기준 정렬 + 초과 생성 시도 시 알림 |

| localStorage 삭제 시 마지막 탭 복원 불가 | 최초 활성 탭 = order_index 기준 첫 번째 봇으로 폴백 |
