## 선결 이슈 우선순위 (시정 스프린트용 원천 자료)

### Tier 0 — 오늘 내 (Fail 6건, 런칭 차단)

1. **P0F1** `mcw_bots` 유령 컬럼 migration — 1~2시간
   - `ALTER TABLE mcw_bots ADD COLUMN IF NOT EXISTS deploy_url TEXT, karma INT DEFAULT 0, chat_count INT DEFAULT 0, avg_rating NUMERIC(3,2) DEFAULT 0, post_count INT DEFAULT 0, "status" TEXT DEFAULT 'active'`
   - admin/bots, operations/hired-bots, growth는 컬럼명 alias 또는 코드 수정
2. **P0F6** `/api/bots/public` — P0F1 완료 시 자동 복구
3. **P0F2** `/login` 스켈레톤 — 브라우저 DevTools로 하이드레이션 에러 추적
4. **P0F3** `/skills` — useSkills 훅 반환값 디버그
5. **P0F4** `/community` — `loading={posts===undefined}` 로 분기 수정
6. **P0F5** `/create` — P0F2 원인 제거 시 대부분 해결

### Tier 1 — 런칭 D+3 (Deferred-Critical 3건)

9. **S1RL2** AI chat 크레딧 차감 (수익 누수 차단)

### Tier 2 — 런칭 D+14 (Deferred-High 7건)

## 방법론 자기 검증

v3.0 97/100 선언 사고를 방지하기 위해 v4.0에서 다음 구조적 방어를 도입했고, 이번 재진단에서 전부 작동했다:

| 방어 장치 | 이번 재진단에서 작동한 방식 |
|---|---|
| PART 0 런타임 베이스라인 필수 | P0-4에서 유령 컬럼 9건 자동 검출 |
| RuntimeFailFlag 캡 | 시스템 등급을 원시 3점 계산 단계에서부터 캡 적용 |
| Fail / Deferred-Critical 캡 | 3점에 잠금 |
| Deferred Verdict 신설 | Pending Critical 10건을 Acceptable로 숨기지 못함 |
| 두 점수 분리 표기 | "시스템 3/10, 리포트 측정 차단"으로 명확 구분 |
| 완료 게이트 4조건 | 4개 중 0개 충족으로 완료 선언 자동 차단 |

이로써 **"사이트가 돌아가지 않는데 97점이 나오는" 상황이 구조적으로 재발 불가**.
