# MCW 디자인 개선방안 연구 보고서
> 작성일: 2026-04-08
> 기준: MCW DESIGN.md + globals.css v2.0 vs Vercel/Linear/Stripe/Notion/Supabase

---

## 1. 현재 MCW 디자인 시스템 분석

### 1.1 강점

**색상 시스템 — 명확한 역할 분리**
- 퍼플(`#5E4BFF`) + 앰버(`#F59E0B`) 듀얼 브랜드: AI/수익 역할 분리가 명쾌하다
- CSS 변수 기반 시맨틱 토큰 구조(`rgb(var(--토큰명))`)가 잘 잡혀 있다
- 다크/라이트 양쪽 토큰이 `.dark` / `.light` 클래스로 체계적으로 분리되어 있다
- Raw 팔레트(`--primary-50` ~ `--primary-950`)와 시맨틱 토큰 두 레이어 분리가 올바르다

**타이포그래피 — 한글 최적화**
- Pretendard Variable: 한글 환경에서 최선의 선택. Inter/Geist 대비 한글 가독성 우월
- 400→500→600→700→800 웨이트 시스템이 명확하다
- `line-height: 1.625` — 한글 본문에 적합한 여유로운 값

**애니메이션 시스템**
- `fadeInUp`, `shimmer`, `pulse-glow`, `countUp` 키프레임 내장
- `--transition-fast/base/slow` 트랜지션 토큰 체계화

**그라데이션 프리셋**
- `--gradient-hero-dark`, `--gradient-revenue`, `--gradient-ai` 등 용도별 그라데이션 미리 정의
- Radial gradient hero 배경: Botpress 스타일, 현대적

### 1.2 약점 및 개선 필요 지점

**약점 1 — 보더(Border) 전략의 구체성 부족**
- MCW: `1px solid rgb(var(--border))` — 단순한 솔리드 보더
- 경쟁사들(Linear, Supabase)은 `rgba(255,255,255,0.05~0.08)` 반투명 화이트 보더 또는 Vercel의 "shadow-as-border" 기법 사용
- 현재 MCW는 다크 모드에서 `--border: neutral-700 (#334155)` — 솔리드 불투명 보더로 인해 다크 배경에서 경계선이 다소 거칠게 느껴질 수 있음

**약점 2 — 타이포그래피 letter-spacing 미정의**
- MCW DESIGN.md에 Heading에 `-0.02em` 명시되어 있지만, globals.css에는 display 사이즈별 정밀한 letter-spacing 스케일이 없음
- 경쟁사들: Vercel(-2.4px @48px), Linear(-1.056px @48px), Notion(-2.125px @64px), Stripe(-0.96px @48px)
- MCW는 상대값(`-0.02em`) 하나만 쓰는데, 절대값 픽셀 기반 스케일이 더 정밀한 제어 가능

**약점 3 — 깊이(Depth) 시스템의 단조로움**
- 현재 MCW 그림자: `0 1px 2px 0 rgb(0 0 0 / 0.4)` → `0 10px 15px -3px rgb(0 0 0 / 0.6)` — 단순 흑색 그림자
- Stripe: blue-tinted `rgba(50,50,93,0.25)` — 브랜드 색상이 깊이까지 관통
- Vercel: inner `#fafafa` ring + multi-layer stack — 카드가 내부에서 빛나는 느낌
- MCW의 퍼플 글로우가 있지만 CTA 버튼에만 제한됨. 카드 호버에도 확장 가능

**약점 4 — 라이트 모드 깊이감 부재**
- 라이트 모드 `--shadow-sm: 0 1px 2px 0 rgb(0 0 0 / 0.05)` — 너무 약함
- Notion의 5-layer 누적 그림자(`max opacity 0.05 per layer`)처럼 여러 레이어가 쌓여야 "질감 있는 라이트 카드" 느낌

**약점 5 — 버튼 radius 일관성**
- 경쟁사: Vercel/Linear/Stripe = 6px (subtle), Notion = 4px
- MCW는 `--radius-lg (12px)` — 경쟁사 대비 과도하게 둥근 버튼
- 챗봇 플랫폼이라 "친근함" 컨셉에는 맞지만, 프리미엄/기술적 인상 희석

**약점 6 — 색상 온도 불일치**
- MCW 다크 배경: Slate계열 (`#0F172A`) — 블루 톤 → cool
- 앰버 골드(`#F59E0B`) — warm → 두 온도가 충돌
- 경쟁사들은 온도를 통일: Linear(cool violet), Stripe(cool navy+warm purple), Notion(warm warm warm)

| 항목 | MCW (다크) | Vercel | Linear | Stripe | Notion |

## 3. 구체적 개선안

### 3.1 Quick Win (즉시 적용 가능, CSS 변수 수정만 필요)

#### QW-1: 다크 모드 보더 — 반투명 화이트로 전환

**현재**:
```css
--border: var(--neutral-700);  /* #334155 — 솔리드, 불투명 */
```

**개선**:
```css
--border:        255 255 255 / 0.08;  /* rgba(255,255,255,0.08) — Linear 스타일 */
--border-subtle: 255 255 255 / 0.05;  /* rgba(255,255,255,0.05) */
--border-strong: 255 255 255 / 0.12;  /* rgba(255,255,255,0.12) */
```

**효과**: 다크 배경에서 보더가 "공중에 떠있는" 느낌 대신 배경과 자연스럽게 섞임. Linear, Supabase의 세련됨 획득. 단, 라이트 모드는 현행 유지(`rgba(0,0,0,0.1)` 화이트 배경 대응).

---

#### QW-2: Hero 타이포그래피 letter-spacing — 픽셀 스케일 추가

**현재**:
```css
h1, h2, h3, h4, h5, h6 {
  letter-spacing: -0.02em;  /* 상대값 하나만 */
}
```

**개선** (유틸 클래스로 추가):
```css
/* Display Hero (48px): -0.02em ≈ -0.96px → 정밀하게 */
.text-hero    { letter-spacing: -1.5px; }   /* 48px에서 */
.text-display { letter-spacing: -1.0px; }   /* 36px에서 */
.text-h1      { letter-spacing: -0.6px; }   /* 30px에서 */
.text-h2      { letter-spacing: -0.3px; }   /* 24px에서 */
/* 20px 이하는 normal */
```

**효과**: Vercel/Linear급 타이포그래피 정밀도. 헤딩이 더 "엔지니어드" 느낌.

---

#### QW-3: 라이트 모드 카드 그림자 — Notion 5-layer 방식 적용

**현재**:
```css
/* 라이트 */
--shadow-md: 0 4px 6px -1px rgb(0 0 0 / 0.1), 0 2px 4px -2px rgb(0 0 0 / 0.1);
```

**개선**:
```css
--shadow-md:
  0 1px 3px 0 rgb(0 0 0 / 0.04),
  0 2px 8px 0 rgb(0 0 0 / 0.03),
  0 4px 18px 0 rgb(0 0 0 / 0.04),
  0 8px 32px 0 rgb(0 0 0 / 0.02);

--shadow-lg:
  0 2px 4px 0 rgb(0 0 0 / 0.04),
  0 4px 12px 0 rgb(0 0 0 / 0.04),
  0 8px 24px 0 rgb(0 0 0 / 0.05),
  0 16px 48px 0 rgb(0 0 0 / 0.03),
  0 24px 64px 0 rgb(0 0 0 / 0.02);
```

**효과**: 라이트 카드가 "질감 있게" 페이지에 안착. 현재 단조로운 single-layer 그림자 대비 깊이감 대폭 향상.

---

#### QW-4: 다크 모드 카드 그림자 불투명도 낮추기

**현재**:
```css
--shadow-sm:  0 1px 2px 0 rgb(0 0 0 / 0.4);   /* 0.4 — 과함 */
--shadow-md:  0 4px 6px -1px rgb(0 0 0 / 0.5); /* 0.5 — 과함 */
--shadow-lg:  0 10px 15px -3px rgb(0 0 0 / 0.6);
--shadow-xl:  0 20px 25px -5px rgb(0 0 0 / 0.7);
```

**개선**:
```css
--shadow-sm:  0 1px 2px 0 rgb(0 0 0 / 0.25);
--shadow-md:  0 4px 6px -1px rgb(0 0 0 / 0.3), 0 2px 4px -2px rgb(0 0 0 / 0.2);
--shadow-lg:  0 10px 15px -3px rgb(0 0 0 / 0.35), 0 4px 6px -4px rgb(0 0 0 / 0.2);
--shadow-xl:  0 20px 25px -5px rgb(0 0 0 / 0.4), 0 8px 10px -6px rgb(0 0 0 / 0.25);
```

**효과**: 다크 모드 카드가 덜 "무겁게" 느껴짐. Linear 수준의 세련된 다크 카드 획득.

---

#### QW-5: 퍼플 글로우 브랜드 그림자 — 카드 호버에 확장

**현재**: CTA 버튼에만 퍼플 글로우 사용

**개선** (별도 유틸 클래스):
```css
/* 카드 호버 시 퍼플 보더 + 미묘한 글로우 */
.card-hover:hover {
  border-color: rgb(var(--primary-400) / 0.5);
  box-shadow:
    0 0 0 1px rgb(var(--primary-400) / 0.2),
    0 8px 24px rgb(var(--primary-500) / 0.1),
    var(--shadow-lg);
}
```

**효과**: Stripe의 "브랜드 색상이 그림자까지 관통" 철학과 일치. MCW 퍼플 아이덴티티 강화.

---

#### QW-6: Pretendard OpenType Feature 활성화

**현재**: OpenType feature 없음

**개선**:
```css
body {
  font-feature-settings: "liga" 1, "kern" 1;
}

/* 숫자 테이블에서 */
.tabular-nums {
  font-feature-settings: "tnum" 1;
}
```

**효과**: 리가처(글자 연결) 활성화로 타이포그래피 세련도 향상. `"tnum"`으로 수익/크레딧 숫자 정렬 개선.

---

### 3.2 중기 개선 (디자인 리뉴얼 수준, 컴포넌트 재작성 필요)

#### MR-1: 버튼 radius 조정 — 더블 트랙 시스템

현재 MCW: 모든 버튼 `--radius-lg (12px)` → 경쟁사 4-6px 대비 과도

**제안: 맥락별 radius 분기**
```
Primary CTA (랜딩, 강한 CTA): 12px 유지 → 친근한 브랜드 이미지 강조
App 내부 UI 버튼 (관리, 설정): 6-8px → 더 professional한 인상
Pill 버튼 (필터, 태그, 뱃지): 9999px → Supabase 스타일 분명한 차별화
```

**효과**: 랜딩은 친근하게, 앱 내부는 프로페셔널하게. 맥락별 버튼 언어 차별화.

---

#### MR-2: 카드 깊이 시스템 — "Floating vs Embedded" 이중 레벨

**현재**: 모든 카드가 동일한 깊이

**제안**:
```css
/* Level 1 — Embedded (페이지에 안착): 리스트 카드 */
.card-embedded {
  background: rgb(var(--bg-surface));

  box-shadow: var(--shadow-sm);
```

### 3.3 장기 개선 (풀 리디자인 수준)

#### LT-1: 디자인 철학 선택 — "두 가지 길"

현재 MCW는 "Linear의 정밀한 다크 UI + Stripe의 프리미엄 수익 UI 결합"을 지향한다고 DESIGN.md에 명시되어 있다. 그러나 이 두 방향은 실제로 긴장 관계에 있다:

| | Linear 방향 | Stripe 방향 |
|-|------------|------------|
| 배경 | 극도로 어두운(#08090a) | 화이트 라이트(#ffffff) |
| 색상 | 단색(인디고 하나) | 퍼플 + 루비 + 마젠타 |
| 타이포 | 가는 두께(510) | 극도로 가는(300) |
| 분위기 | 개발자 도구, 생산성 | 금융, 프리미엄 소비재 |

**MCW 추천 방향**: "Linear 기반 + Stripe 수익 요소 선택적 도입"
- 전체 구조/색상/깊이: Linear
- 수익/결제/CTA 섹션: Stripe의 blue-tinted shadow + dark brand section 방식
- 이 두 영역을 명확히 분리하면 "기술적이면서도 프리미엄" 실현 가능

---

#### LT-2: 색상 시스템 — HSL 토큰으로의 전환 검토

**현재**: RGB 세 값 분리(`94 75 255`) 방식

**Supabase/Radix 방식**: HSL with alpha 토큰
```css
/* HSL 기반 */
--primary-500: hsl(250, 100%, 65%);  /* #5E4BFF */
/* alpha 버전 */
--primary-500-a20: hsla(250, 100%, 65%, 0.2);
```

**장점**: HSL은 인간이 직관적으로 편집하기 좋음. Lightness 조정만으로 다크/라이트 변환 가능.
**단점**: MCW 현재 `rgb(var(--토큰) / 0.2)` 패턴이 이미 알파 조합을 지원. 마이그레이션 비용 있음.
**결론**: 장기적으로는 HSL 전환이 유리하나, 현재 RGB 방식도 충분히 기능적. 우선순위 낮음.

---

#### LT-3: 컴포넌트 라이브러리 — Radix UI 프리미티브 기반 재구축

Linear, Supabase가 Radix UI 프리미티브(접근성 내장 컴포넌트) 위에 스타일링하는 방식 채택.

**현재**: Tailwind 유틸리티 클래스만 사용 추정

**개선**: Radix UI + Tailwind CSS 조합
- Dialog, Dropdown, Tooltip, Popover — Radix 프리미티브 사용
- 접근성(WAI-ARIA) 자동 충족
- 포커스 관리, 키보드 탐색 자동화

**효과**: 접근성 AA/AAA 준수, 모달/드롭다운 애니메이션 품질 향상.

---

#### LT-4: 모션 시스템 고도화

**현재**: 4개 keyframe + 3개 transition 변수

**개선**: 의미론적 모션 토큰 시스템
```css
/* 목적별 easing */
--ease-appear:   cubic-bezier(0.16, 1, 0.3, 1);   /* spring-like, 나타날 때 */
--ease-disappear: cubic-bezier(0.7, 0, 1, 0.3);   /* 빠른 퇴장 */
--ease-move:     cubic-bezier(0.4, 0, 0.2, 1);     /* 요소 이동 */
--ease-bounce:   cubic-bezier(0.68, -0.55, 0.265, 1.55); /* 스프링 */

/* 지속시간별 */
--duration-instant: 50ms;
--duration-fast:   150ms;
--duration-normal: 250ms;
--duration-slow:   400ms;
--duration-slower: 700ms;
```

**효과**: 일관된 모션 언어. "이 사이트의 느낌"을 모션이 강화.

### 핵심 결론

MCW 디자인 시스템의 **기반(토큰 구조, 듀얼 브랜드, 다크/라이트 시스템)은 탄탄하다.** 문제는 세부 튜닝 — 특히 보더 투명도, 그림자 불투명도, letter-spacing 스케일 — 에서 경쟁사 대비 거친 부분이 남아 있다.

Quick Win 6개를 적용하면 globals.css 변수 조정만으로 **Linear/Notion 수준의 섬세함**을 획득할 수 있다. 컴포넌트 재작성 없이 즉각적인 품질 향상이 가능하다는 점이 MCW 현재 시스템의 가장 큰 강점이다.
