## 1. 레이아웃 — 3 Viewport

### 1.1 Desktop (≥1440)

```
┌─────────────────────────────────────────────────────────────────┐
│  GNB  [CoCoBot] [홈] [Birth] [Skills] [Jobs] [Community]   👤▼ │
│                                           └ 드롭다운: 내 봇 · /hub
├─────────────────────────────────────────────────────────────────┤
│ ┌─────────────────────── PersonaTabBar ───────────────────────┐ │
│ │ [🤖 알파] [📚 베타] [🎨 감마] [💼 델타] [...▼]  [+]         │ │
│ │   ▔▔▔▔▔                                                     │ │
│ └─────────────────────────────────────────────────────────────┘ │
│                                                                 │
│ ┌─────────────────── ChatWindow (active tab) ────────────────┐ │
│ │                                                             │ │
│ │   봇 알파와의 대화 (conv_id: uuid, messages: [...])          │ │
│ │                                                             │ │
│ │                                                             │ │
│ │                                          [ 메시지 입력... ] │ │
│ └─────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────┘
```

- GNB 기존 4대 메뉴(Birth/Skills/Jobs/Community) **무손상 유지** (feedback_no_invented_nav)
- 포털 진입은 우측 프로필 드롭다운 "내 봇" 링크 → `/hub`
- 탭바: 가로 flex, 활성 탭 밑줄 2px 퍼플, 비활성 텍스트 60% opacity
- `+` 탭 항상 우측 끝, 10개 도달 시 disabled

### 1.2 Tablet (768–1023)

- GNB 동일
- 탭바 가로 flex 유지 (tablet 은 보통 6~7개 정상 노출)
- ChatWindow padding 축소 (16px)

### 1.3 Mobile (390)

```
┌─────────────────────────┐
│ ☰ CoCoBot         👤▼  │
├─────────────────────────┤
│ ┌─ Sticky TabBar ─────┐ │
│ │→[🤖알파][📚베타][...] │ │ ← 가로 스와이프
│ │   ▔▔▔▔▔              │ │
│ └──────────────────────┘ │
├─────────────────────────┤
│                         │
│   ChatWindow            │
│   (full-width)          │
│                         │
│   [ 메시지 입력... ]    │
└─────────────────────────┘
```

- 탭바: `sticky top-0 z-20`, `overflow-x-auto scroll-snap-x scroll-snap-mandatory`
- 탭 1개 min-width: 120px (아바타 32 + 이름 truncate)
- 활성 탭 `scrollIntoView({ inline: 'center' })`
- 페이지 자체 가로 스크롤 0 (탭바 내부만 스크롤)
- 모든 터치 대상 ≥44px

## 2. 상태 전이 다이어그램

### 2.1 탭 라이프사이클

```
            +────────── onSelect(botId) ────────+
            │                                    │
            ▼                                    │
   ┌─────────────────┐                           │
   │  INACTIVE       │  (state in Context.Map)   │
   │  (unmounted)    │                           │
   └────────┬────────┘                           │
            │ activate                           │
            ▼                                    │
   ┌─────────────────┐   탭 이탈                 │
   │  MOUNTING       │ ─── abort() ──────────────┘
   │  React.lazy     │
   └────────┬────────┘
            │ Suspense resolved
            ▼
   ┌─────────────────┐
   │  ACTIVE         │ ← 사용자 상호작용 (입력/스크롤/스트림)
   │  (scrollTop,    │
   │   messages,     │
   │   inputDraft,   │
   │   streaming)    │
   └─────────────────┘
```

### 2.2 스트리밍 중 탭 이탈

```
ACTIVE (streaming=true)
   │
   │ onSelect(otherBotId)
   ▼
abortController.abort()                      ← SSE 취소
   │
   ▼
TabState 스냅샷 저장 (messages up-to-now, streaming=false)
   │
   ▼
INACTIVE → 다른 탭 MOUNTING
```

- 재진입 시 재스트리밍 불가 → "대화가 중단되었습니다. 마지막 메시지를 다시 전송하세요." 토스트

### 2.3 + 탭 → Birth 위저드 완료

```
ACTIVE tab X
   │
   │ onAdd() [10개 미만]
   ▼
Birth 모달 오픈 (Step1~Step8)
   │
   │ Step8 완료 → /api/bots refetch
   ▼
bots[] 에 newBot push (order_index = bots.length)
   │
   ▼
setActive(newBot.id) → 새 탭 활성 MOUNTING
```

## 3. 핵심 인터랙션 & 사용자 편의성

| 기능 | 해결책 | 사용자 편의 효과 |
|------|--------|-----------------|
| 여러 봇 공존 | 가로 탭바 (최대 10) | 재로딩/이탈 없이 즉시 전환 |
| 탭 많을 때 인지 과부하 | 10 하드리밋 + `...▼` | 시야 집중 + 나머지 접근성 보존 |
| 대화 상태 휘발 | Context.Map 스냅샷 | 100% 복원, 중단 없음 |
| 딥링크 공유 | `/hub?tab=botId` | URL 하나로 특정 페르소나 직행 |
| 마지막 본 탭 기억 | localStorage | 재방문 시 즉시 그 탭 활성 |
| 모바일 작은 화면 | sticky + 가로 스와이프 | 엄지 한 손 조작 가능 |
| 신규 페르소나 | `+` 탭 → 모달 위저드 | 포털 이탈 없이 추가 |

## 4. 접근성 & 성능 KPI 반영

- `role="tablist"` / `role="tab"` / `aria-selected` / `aria-disabled`
- 탭 간 `←/→` 키 이동, `Home/End` 지원
- 탭 전환 <150ms (번들 분할 + lazy mount)
- 활성 탭만 마운트 (DOM 1/N)
- Lighthouse Mobile ≥80 (S11 KPI 승계)
