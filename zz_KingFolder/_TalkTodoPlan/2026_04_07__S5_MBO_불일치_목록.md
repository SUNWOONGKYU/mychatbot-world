## N. 추가 발견 불일치 (29페이지 전수 스크린샷 기반)

### N1. /home — ~~완전 빈 화면~~ **[오진 — 비로그인 시 /login 리다이렉트 정상 동작]**
- 코드 검증: `app/home/page.tsx`는 6탭 대시보드 (프로필/챗봇/스킬/수익/크레딧/보안)
- 비로그인 시 `router.replace('/login')` → headless 스크린샷이 리다이렉트 전 빈 화면 캡처한 것
- **실제 문제**: /home과 /mypage 역할 중복 → 하나로 통합 필요 (라우트 정리 P 섹션에서 처리)

### N2. /birth — ~~완전 빈 화면~~ **[오진 — ?botId= 파라미터 필수]**
- 코드 검증: `app/birth/page.tsx`는 `?botId=` 없으면 `/home` 리다이렉트
- 챗봇 생성 완료 후 탄생 애니메이션 페이지 — 파라미터 없이 직접 접근 시 빈 화면은 정상
- /birth와 /create 관계: /create(위저드) 완료 → /birth?botId=xxx (탄생 축하) — 정상 흐름

### N3. /login — 디자인 시스템 미적용
- Google/카카오 소셜 로그인 버튼 존재 — 기능은 정상
- 상단바(Navbar) 없음 — 로그인 페이지에서 사이트 네비게이션 불가
- S5 디자인 토큰(퍼플/앰버) 미적용 — 기본 Tailwind 스타일
- 카카오 버튼: 노란색 — 맞지만, Google 버튼과 크기/정렬 불일치

### N4. /signup — 디자인 시스템 미적용
- 상단바 없음
- S5 컬러 토큰 미적용 — 기본 퍼플 버튼만
- 폼 카드 라운딩/그림자가 P3 스펙과 다름

### N5. /reset-password — 디자인 시스템 부분 적용
- 상단바는 있음 (앱 Navbar)
- S5 퍼플 버튼 적용됨
- 하지만 /login, /signup에는 상단바가 없어서 불일치

### N6. /marketplace — /skills와 중복
- /marketplace 페이지 존재 (스킬 마켓플레이스 검색+카드)
- /skills 페이지도 존재 (스킬장터)
- 두 페이지가 유사한 기능 → 어떤 걸 사용할지 정리 필요
