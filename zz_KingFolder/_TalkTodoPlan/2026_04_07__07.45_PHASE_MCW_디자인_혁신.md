## Task 상세

### P1. 네비게이션 구조 설계

**목표**: Vanilla 원본 5대 메뉴 기반 + 벤치마킹 결과 반영

**결정 사항**:
- 랜딩: 상단바 (Logo + 5대메뉴 + 로그인 + CTA)
- 앱 내부: 상단바 유지 or 좌측 사이드바?
- 모바일: 하단 탭바 5개
- 메뉴 구성:
  - Vanilla 원본: Birth(탄생) / Learning(학습) / Skills(스킬장터) / Jobs(구봇구직) / Community(봇카페)
  - 추가 검토: Home(대시보드)은 별도? 마이페이지는?

**산출물**: 네비게이션 구조도 (데스크탑/태블릿/모바일)

---

### P2. 컬러 시스템 + 디자인 토큰 정의

**목표**: "AI 챗봇 수익 플랫폼" 정체성을 담은 컬러 체계

**벤치마킹 기반**:
- Primary: 딥 퍼플 (#5E4BFF 계열) — AI/기술 신뢰감
- Accent: 앰버/골드 (#F59E0B 계열) — 수익/크리에이터
- Background: 다크 모드 퍼스트 (Botpress/Copy.ai 스타일)
- 그라데이션: 퍼플→블루(AI), 앰버→오렌지(수익)

**산출물**:
- CSS 변수 팔레트 (--color-primary, --color-accent 등)
- 라이트/다크 모드 토큰
- 타이포그래피 스케일 (Pretendard or Noto Sans KR)
- 간격/라운딩/그림자 토큰

---

### P3. 핵심 컴포넌트 디자인 스펙

**목표**: 반복 사용되는 UI 컴포넌트의 디자인 규격

**대상 컴포넌트**:
1. Button (Primary/Secondary/Ghost/Danger)
2. Card (스킬카드, 봇카드, 잡카드, 포스트카드)
3. Input (텍스트, 검색, 셀렉트, 체크박스)
4. Badge (카테고리, 가격, 상태, NEW)
5. Tab (필터탭, 네비탭)
6. Modal (확인, 폼, 알림)
7. Toast (성공, 에러, 정보)
8. 스켈레톤 로딩 카드
9. Hero 섹션 (그라데이션 배경 + 타이틀 + CTA)
10. 봇 아바타/프로필 카드

**산출물**: 컴포넌트별 디자인 스펙 (크기, 색상, 상태별 변화)

---

### P4. 페이지별 와이어프레임/레이아웃

**목표**: 각 페이지의 섹션 구성 + 정보 배치 확정

**대상 페이지** (우선순위순):
1. 랜딩 (/) — Hero + 데모 + 챗봇유형 + 가격 + 푸터
2. 챗봇 만들기 (/create) — 스텝 위저드 8단계
3. 챗봇 대화 (/bot/[id]) — 채팅 UI
4. 스킬장터 (/skills) — 카드 그리드 + 필터
5. 구봇구직 (/jobs) — 검색 + 탭 + 카드
6. 학습 (/learning) — 커리큘럼 카드 + 진행률
7. 봇카페 (/community) — 피드 + 사이드바
8. 홈 대시보드 (/home) — 탭 기반 관리
9. 게스트 모드 (/guest) — 카테고리 카드

**산출물**: 페이지별 섹션 배치도 (텍스트 와이어프레임)
