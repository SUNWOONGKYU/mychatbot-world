# AI 팀 구성 및 운영 가이드

## 🤖 AI 팀 개요

우리 프로젝트는 인간 최소화, AI 최대화 전략을 채택합니다.  
실제 개발의 80-90%는 AI 에이전트들이 수행하며, 인간은 전략적 의사결정과 품질 검증에 집중합니다.

## 🔄 협업 프로세스

### 1. 단순 작업 (순차 처리)
```
사용자 요청
    ↓
써니봇2 분석
    ↓
단일 에이전트 실행
    ↓
결과 제공
```

**예시:** 간단한 FAQ 생성
```
gemini "고객 서비스 챗봇 FAQ 10개 생성"
```

---

### 2. 중간 복잡도 (병렬 처리)
```
사용자 요청
    ↓
써니봇2 작업 분해
    ↓
┌─────────┬─────────┬─────────┐
│Agent A  │Agent B  │Agent C  │ (병렬)
└─────────┴─────────┴─────────┘
    ↓
써니봇2 결과 통합
    ↓
최종 산출물
```

**예시:** 챗봇 1개 개발
```python
# 동시 실행
sessions_spawn(task="페르소나 개발", label="persona")
sessions_spawn(task="백엔드 API", label="backend")
sessions_spawn(task="프론트엔드 UI", label="frontend")

# 써니봇2가 모두 완료되면 통합
```

---

### 3. 고복잡도 (계층적 병렬)
```
사용자 요청
    ↓
써니봇2 (전략 수립)
    ↓
Claude Opus (아키텍처 설계)
    ↓
┌──────────┬──────────┬──────────┐
│기획 병렬 │개발 병렬 │콘텐츠병렬│
└──────────┴──────────┴──────────┘
    ↓         ↓         ↓
    └─────────┴─────────┘
            ↓
    통합 & 테스트 (병렬)
            ↓
        최종 검수
```

**예시:** 전체 시스템 개발
```python
# 1단계: 아키텍처
arch = sessions_spawn(task="전체 아키텍처 설계", model="opus")

# 2단계: 병렬 개발 (아키텍처 완료 후)
sessions_spawn(task="백엔드 개발", label="be")
sessions_spawn(task="프론트엔드 개발", label="fe")
sessions_spawn(task="페르소나 개발", label="persona")
sessions_spawn(task="문서 작성", label="docs")

# 3단계: 통합 & 테스트
sessions_spawn(task="통합 테스트", label="test")
```
