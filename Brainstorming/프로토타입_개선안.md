## 1. 현황 분석

### 1-1. 현재 구현된 기능 목록

#### 챗봇 생성 (Birth)
- 8단계 생성 위저드: 기본정보 → 페르소나 → 인터뷰 → AI분석 → 생성 → 아바타 → 테마 → 배포
- 음성 인터뷰(STT: Whisper API) + 텍스트 입력 병행
- AI 자동 분석: 인사말 + FAQ 5개 자동 생성 (`api/create-bot.js`)
- 페르소나별 IQ/EQ 슬라이더 (감성-논리 밸런스 설정)
- 음성 선택: 6종 (Fable, Nova, Shimmer, Alloy, Echo, Onyx)
- 아바타 이모지 선택 + 이미지 업로드
- 테마 선택: 다크/라이트 모드, 5가지 컬러
- 배포 채널 선택: 웹, 카카오톡, 텔레그램
- URL/QR 코드 자동 생성

#### 챗봇 대화 (Chat)
- TTS 출력: OpenAI TTS-1 → SpeechSynthesis → Google Translate 3단 폴백 (→ unspeech 통합 예정)
- STT 입력: Whisper API (MediaRecorder + 4초 무음 자동 중지 → VAD 교체 예정)
- 페르소나 전환 (대면용 avatar / 내부용 helper 구분)
- SSE 스트리밍 응답 (streaming fallback → non-streaming)
- 대화 이력 저장: localStorage + Supabase `mcw_chat_logs`
- DM 보안: public / allowlist / pairing 3가지 정책
- CPC 소대 연동 (claude.ai 원격 제어) — `Claude 연락병` / `Trade 연락병` 두 가지 페르소나 지원

#### AI 모델 라우팅
- OpenRouter 경유 다중 모델 폴백 (`MODEL_STACK`)
- 현재 스택: `google/gemini-2.5-flash` → `openai/gpt-4o` → `anthropic/claude-sonnet-4.5` → `deepseek/deepseek-chat`
- 다중 API 키 로테이션 + 30초 쿨다운
- 컨텍스트 오버플로우 감지 → 자동 요약 압축

#### 지식베이스 (RAG)
- Obsidian 지식베이스 연동 (pgvector 유사도 검색)
- 페르소나별 지식 범위 격리
- 문서 업로드/웹 크롤링 스킬 (`pdf-upload`, `web-crawl`)

#### 스킬 시스템
- 스킬 카탈로그 API (`/api/skills`)
- 현재 스킬 22개: reservation, survey, coupon, lead-collect, google-cal, email-send, kakao-noti, 3d-avatar, custom-theme, emoji-react, faq-auto, multilang, pdf-upload, profanity-filter, sentiment, spam-block, stats-analysis, trader-expert, tts-basic, voice-clone, web-crawl, backup
  - ※ 스킬 카탈로그 목록에서 `email-send` 중복 제거됨 (실제 22개)
- 스킬 외부 API 실행 (`/api/skill-integrations`)
- 페르소나별 스킬 독립 관리

#### 배포/채널
- Vercel Serverless (웹)
- 텔레그램 봇 연동 (`/api/telegram.js`)
- 채널 어댑터 구조 (`js/channel-adapter.js`)

#### UI/UX (css/chat.css)
- 다크/라이트 모드 전환 구현 (localStorage 저장)
- 모바일 기반 CSS 구현: `100dvh`, `safe-area-inset-bottom`, `@media (max-width: 600px)` 반응형
- 페르소나 칩 모바일 4열 자동 배치
- 단, PWA 미지원, 음성 버튼 터치 영역 개선 여지 있음, 카카오톡 채널 연동 UI만 있고 실제 로직 없음

## 3. 개선 로드맵

### Phase 1 — MVP (사업비 투입 후 2개월 내) · 예상 공수: 개발자 1인 기준 약 6주
> 핵심 가치 1(접근성) + 2(최소 비용) + 3(쉬움) 기반 다지기

**목표**: 누구나 무료로, 쉽게 챗봇을 만들고 즉시 배포

- [ ] 게스트 체험 모드 (회원가입 없이 데모 생성)
- [ ] 무료 AI 모델 우선 라우팅 (Gemini Flash Free, Llama 3 Free 등)
- [ ] 무료 플랜 명시 + 사용량 한도 UI
- [ ] 모바일 UX 전면 개선 (반응형 레이아웃, 터치 최적화)
- [ ] 소셜 로그인 (카카오, 구글)
- [ ] 음성 확인 절차: 챗봇 생성 전 "생성된 인사말을 음성으로 확인하세요" 단계
- [ ] 온보딩 튜토리얼 (첫 로그인 시 안내)
- [ ] 카카오톡 채널 연동 실제 구현 (현재 UI만 있고 로직 없음)

### Phase 2 — 베타 (3~4개월) · 예상 공수: 개발자 1인 기준 약 5~6주
> 핵심 가치 4(성장) 구현

**목표**: 챗봇이 교육받고 성장하는 플랫폼 완성

- [ ] 챗봇스쿨: 시나리오 훈련 콘텐츠 (까다로운 고객, 긴급 상황 등)
- [ ] AI 멘토링: 대화 품질 자동 분석 + 개선 제안
- [ ] 성장 지표 대시보드: 대화 수, 만족도, 응답 품질 점수
- [ ] 챗봇 레벨 시스템 (Lv.1 초보 → Lv.10 전문가)
- [ ] 챗봇 커뮤니티: 노하우 공유, 챗봇 간 협업 프로토콜
- [ ] 스킬 마켓플레이스 기초 (사용자 스킬 업로드/공유)
- [ ] 학습 포트폴리오 (어떤 교육을 받았는지 이력 관리)

### Phase 3 — 정식 출시 (5~7개월) · 예상 공수: 개발자 1인 기준 약 7~8주
> 핵심 가치 5(수익 창출) + 6(상속) 구현

**목표**: 챗봇이 수익을 창출하고 자산으로 상속되는 완전한 생태계

- [ ] 스킬 마켓플레이스 정식 오픈 (유료 스킬 거래, 수수료 정산)
- [ ] 챗봇 중개 수익 모델: 예약/리드/쿠폰 → 성과 기반 수수료
- [ ] 비즈니스 대시보드: 수익 현황, 성과 분석, 세금 계산서
- [ ] 상속 시스템: 상속 대상 지정, 디지털 자산 패키지 생성
- [ ] 상속 접근 이전 절차 (법적 효력 있는 디지털 유언 연계)
- [ ] API 오픈: 챗봇 기능 외부 연동 (파트너사 API)
- [ ] 엔터프라이즈 플랜 (팀 단위 관리, SLA)

### 4-2. 최소 비용 (무료 우선 라우팅) 구현

**현재**: `MODEL_STACK`에서 `gemini-2.5-flash` 1순위지만 유료 모델. OpenRouter 직접 호출 방식으로 모델 전환 시 코드 수정 필요

**개선 방향**:

**xsai SDK 도입** (참고: `참고자료_AIRI/xsai/`, MIT, 1.7KB 초경량):
- 현재 OpenRouter 직접 `fetch()` 호출 → xsai SDK로 교체
- 감성 슬라이더(IQ/EQ) → `temperature` 매핑, 비용 슬라이더 → 모델 자동 선택
- 모델 전환 시 코드 수정 없이 설정만 변경 가능

```javascript
// 현재: OpenRouter 직접 호출 (api/_shared.js)
const res = await fetch('https://openrouter.ai/api/v1/chat/completions', {
  headers: { 'Authorization': `Bearer ${key}` },
  body: JSON.stringify({ model: MODEL_STACK[i], messages, temperature })
});

// 개선: xsai SDK 활용 (동일 OpenRouter 경유, 코드 간결화)
import { generateText } from '@xsai/generate-text';

const result = await generateText({
  provider: openRouter,          // OpenRouter 프로바이더
  model: getOptimalModel(emotionLevel, costLimit),  // 감성+비용 기반 자동 선택
  messages,
  temperature: emotionLevel / 100,  // 감성 슬라이더 직접 매핑
});

// 개선된 MODEL_STACK (계층화)
export const MODEL_STACK_FREE = [
  'google/gemini-2.0-flash-exp:free',   // 완전 무료 (OpenRouter)
  'meta-llama/llama-3.1-8b-instruct:free', // 완전 무료
  'mistralai/mistral-7b-instruct:free', // 완전 무료
];

export const MODEL_STACK_PAID = [
  'google/gemini-2.5-flash',
  'openai/gpt-4o',
  'anthropic/claude-sonnet-4.5',
  'deepseek/deepseek-chat',
];

// 감성+비용 기반 최적 모델 선택 (특허 도 4 라우팅부 구현)
export function getOptimalModel(emotionLevel, costLimit, userPlan = 'free') {
  const candidates = userPlan === 'free'
    ? [...MODEL_STACK_FREE, ...MODEL_STACK_PAID]
    : MODEL_STACK_PAID;
  // 비용 한도 내 필터링 → 감성 적합도 최고 모델 자동 선택
  return filterByCost(candidates, costLimit)
    .sort((a, b) => emotionFit(b, emotionLevel) - emotionFit(a, emotionLevel))[0];
}
```

**추가 구현**:
- 월 사용량 대시보드 (무료 플랜: 월 500회, 유료: 무제한)
- 사용량 초과 시 자동 알림 + 업그레이드 안내
- 구독료 없음 명시 — OpenRouter 키 직접 연결 옵션 제공 (Power User)
- xsai 도입 시 프로바이더 전환만으로 OpenRouter 외 직접 API 호출도 가능 (비용 절감)

### 4-3. 쉬움 (UX 개선)

**현재**: 음성 인터뷰는 있으나 결과 확인 절차 미흡. STT 입력 후 자동 전송 없이 사용자가 직접 확인 후 전송하는 구조 (의도적 설계, `chat.js` 주석 참조) — 이는 모바일 AudioContext 자동재생 정책 회피를 위한 기술적 이유. 현재 STT는 4초 무음 타이머 기반으로 발화 경계를 감지하는데, 정확도가 떨어짐

**개선 방향**:

```
[음성 입력 개선 — VAD 파이프라인 도입]
참고: 참고자료_AIRI/webai-example-realtime-voice-chat/ (MIT)

현재: MediaRecorder + 4초 무음 타이머 → Whisper API
개선: VAD(Voice Activity Detection) → 정확한 발화 경계 감지 → Whisper API
     → 발화 끝을 자동 감지하여 자연스럽게 STT 전송

구현 방법:
- @ricky0123/vad-web 라이브러리 (브라우저 내 ONNX 모델)
- 사용자가 말을 멈추면 자동으로 STT 전송 (4초 고정 → 실제 발화 끝 감지)
- 5분 음성 인터뷰에서 특히 효과적 (긴 침묵 vs 짧은 쉼 구분)

[음성 확인 절차 — Step 4 개선]
현재: AI 분석 → 텍스트로 FAQ 표시 → "다음" 버튼
개선: AI 분석 → "생성된 인사말을 들어보세요" → TTS 재생 버튼
     → "이 인사말이 맞으시면 확인" → 승인/수정 선택

[진행 가이드]
- 각 단계마다 "왜 이 정보가 필요한가?" 간단 설명 추가
- 예시 완성 챗봇 미리보기 (실제 대화 화면 팝업)

[입력 간소화]
- Step 2 페르소나 설정: "AI가 추천해드릴까요?" 버튼 추가
  → 인터뷰 내용 없어도 직업 선택만으로 자동 생성

[오류 메시지 개선]
- 현재: alert() 팝업
- 개선: 인라인 오류 표시 + 구체적인 해결 방법 안내
```

          │         ↕
