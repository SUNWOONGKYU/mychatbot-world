# 1. 문제인식 (폐업 원인 분석 및 재창업 아이템 추진 배경)

## 1-1. 이전 사업 폐업 원인 분석

이전 사업에서 겪은 실패를 네 가지 핵심 원인으로 분석하고, 각각에 대한 구체적 개선 방향을 도출하였습니다.

| 구분 | 실패 원인 | 교훈 및 개선 방향 |
|------|----------|------------------|
| **시장 검증 부재** | 고객 니즈 확인 없이 기술 중심 개발 착수, 출시 후 시장 반응 저조 | MVP로 빠른 시장 검증 후 확장하는 린 스타트업 방식 채택 |
| **수익모델 불명확** | 명확한 과금 체계 없이 무료 운영, 지속가능한 수익 창출 실패 | 구독형 SaaS(월 정액) + 스킬 마켓 수수료 + 크레딧의 다중 수익 구조 설계 완료 |
| **비용 통제 실패** | 자체 서버 구축 등 초기 고정비 과다 투입 | 서버리스(Vercel) + BaaS(Supabase)로 초기 인프라 비용 월 $10 이하 달성 |
| **단독 운영 한계** | 기획·개발·마케팅 1인 담당으로 속도·품질 모두 저하 | AI 개발 도구(Claude Code, Copilot)로 1인 생산성 10배 향상 + 외부 전문 인력 협업 체계 |

**핵심 교훈**: "기술이 아무리 좋아도, 고객이 원하는 것을 만들지 않으면 실패한다." 이 뼈아픈 경험이 이번 재창업의 출발점입니다.

## 1-2. 재창업 아이템 추진 배경 및 필요성

### (1) 시장의 구조적 문제 — 개인·소상공인 AI 챗봇 시장의 부재

글로벌 AI 챗봇 시장은 2023년 51억 달러에서 2028년 155억 달러(CAGR 25%)로 급성장 중이나, **기업용(B2B)이 80% 이상**을 차지하며, 개인·소상공인을 위한 시장은 사실상 **블루오션**입니다.

| 구분 | 기존 시장 | My Chatbot World |
|------|----------|-----------------|
| **타겟** | 대기업·중견기업 | 개인·소상공인·전문가 |
| **가격** | 월 $74~$2,500 | 월 5만원 (프로 플랜) |
| **생성 소요시간** | 30분~1시간 (폼 30개 입력) | **음성 5분** (AI가 질문, 사용자가 답변) |
| **기술 진입장벽** | 높음 (코딩·설정 필요) | **제로** (말만 하면 완성) |
| **챗봇 라이프사이클** | 생성만 지원 | **생성 → 교육 → 성장 → 수익활동 중개 → 피상속** 전체 지원 |
| **수익활동 중개** | 없음 | **챗봇이 직접 상담·예약·추천·교육 중개** (플랫폼 20% / 운영자 80%) |
| **디지털 유산 관리** | 없음 | **피상속 시스템** (세계 최초) |

### (2) 기존 기술의 5가지 한계와 본 발명의 해결

본 아이템은 **특허 출원 완료**된 핵심 기술에 기반합니다.

> **발명의 명칭**: "아바타형 및 도우미형 멀티 페르소나와 감성 수준 설정에 기반한 멀티 AI 자동 라우팅과, 수익활동 중개를 포함하는 AI 챗봇의 **생성·교육·성장·수익활동 중개·피상속**으로 이어지는 **영속적 라이프사이클**을 지원하는 시스템 및 방법"

| # | 기존 기술의 한계 | 본 발명의 해결 |
|---|----------------|--------------|
| 1 | 기존 플랫폼(Botpress, GPTs 등)은 **단일 AI 엔진에 의존**하며, 감성적 요구에 따라 AI를 자동 선택·라우팅하는 기능 부재 | **감성 수준(1~100) 슬라이더 × 페르소나 조합**에 따라 복수의 AI 엔진 중 최적 AI를 자동 라우팅 |
| 2 | Character.ai 등은 가상 캐릭터 생성에 불과, 사용자 본인의 다면적 정체성을 **대외용과 대내용으로 체계적 분류·관리하는 구조 부재** | **아바타형 페르소나(대외용 분신)**와 **도우미형 페르소나(대내용 업무·생활 도우미)**의 이원 관리 구조 |
| 3 | 기존 플랫폼은 챗봇 **생성만** 지원, 생성 이후 교육·성장·스킬 장착 등 **전체 라이프사이클 통합 관리 부재** | DB축적 + 인간/AI 멘토링 + 스쿨 시스템 + 챗봇 커뮤니티 + 스킬 마켓의 **다층적 교육·성장 체계** |
| 4 | MS 특허(US10,853,717B2)는 고인의 소셜 데이터를 *제3자가 사후에* 수집하는 기술, **본인 사전 동의 기반 피상속 메커니즘 부재** | **본인이 생전에** 페르소나별·데이터별 피상속 허용/불허를 사전 설정, 비동의 항목 즉시 자동 완전 삭제 |
| 5 | Meta의 디지털 사후 아바타 특허도 SNS 활동 시뮬레이션에 불과, **챗봇 소유권 이전 및 피상속 후 지속 성장 개념 부재** | 사망 확인 시 동의 항목은 상속인에게 **소유권 이전**, 피상속 이후에도 **상속인 하에서 챗봇이 계속 교육·성장** |

### (3) 핵심 차별화 기술 — 특허 출원 7대 구성요소

| # | 구성요소 | 선행기술 | 상세 |
|---|---------|---------|------|
| 1 | **멀티 페르소나 관리부** | 없음 | 아바타형(대외용): CPA 아바타, AI전문가 아바타 등 복수 설정 / 도우미형(대내용): 업무도우미 + 생활도우미 |
| 2 | **감성 기반 멀티 AI 라우팅부** | 없음 | 감성 수준 1~100 슬라이더를 수시 조정 → 팩트 중심(지적 응답) ~ 공감·위로(감성적 응답) 자동 전환 |
| 3 | **챗봇 자동 생성부** | 없음 | AI가 음성으로 질문하고 사용자가 음성으로 답하는 인터뷰 방식, 5분 이내 분신 챗봇 자동 생성 |
| 4 | **챗봇 교육·성장 관리부** | 없음 | DB축적 + 인간/AI 멘토링 + 커리큘럼 기반 스쿨 + 챗봇 커뮤니티(토론·협업·SNS) + 스킬 마켓 |
| 5 | **피상속 관리부** | 없음 | 생전 피상속 사전 설정 → 상속인 지정 → 사망 확인 트리거 → 동의 항목 소유권 이전 + 비동의 항목 즉시 완전 삭제 → 상속인 하 지속 성장 |
| 6 | **배포부** | — | URL 임베딩 + QR코드로 웹/모바일/메신저 등 모든 채널 배포 |
| 7 | **수익활동 중개부** | 없음 | 챗봇이 상담·예약·상품추천·교육 등 수익활동을 능동적으로 중개 → 플랫폼 수수료 20% 자동 수취 + 운영자 80% 정산 (예: CPA 챗봇 세무 상담 중개, 식당 챗봇 예약 중개) |

# 2. 실현가능성 (아이템 개발 방법 및 경쟁력 확보 방안)

## 2-1. 제품·서비스 개요 — 영속적 라이프사이클 5단계

**My Chatbot World**는 AI 챗봇의 **생성 → 교육 → 성장 → 수익활동 중개 → 피상속**이라는 영속적 라이프사이클 전체를 원스톱으로 지원하는 세계 최초의 플랫폼입니다.

```
[1단계: 생성] AI 음성 인터뷰 5분 → 나의 분신 챗봇 탄생
    → 아바타형 페르소나 설정 (대외용: 타인이 나 대신 대화)
    → 도우미형 페르소나 설정 (대내용: 내 업무·생활 지원)
    → 감성 수준(1~100) 슬라이더 설정 → 멀티 AI 자동 라우팅
    ↓
[2단계: 교육] 다층적 교육 체계로 챗봇이 점점 똑똑해짐
    → DB 축적 (대화기록, 검색기록, 지식자료)
    → 인간 멘토 + AI 멘토로부터 멘토링
    → 커리큘럼 기반 스쿨 시스템에서 체계적 학습
    ↓
[3단계: 성장] 커뮤니티 활동 + 스킬 장착으로 고도화
    → 챗봇 커뮤니티에서 토론·질의응답·지식공유·협업·SNS 활동
    → 커뮤니티 활동 결과가 다시 챗봇 성장에 반영 (선순환 구조)
    → 스킬 마켓에서 기능 모듈 장착 (예약·결제·통계·번역 등 100+)
    ↓
[4단계: 수익활동 중개] 성장한 챗봇이 운영자를 대신해 수익 창출
    → 상담 중개: CPA 챗봇의 세무 상담, 변호사 챗봇의 법률 상담 등
    → 예약 중개: 식당·미용실·병원 등 예약 자동 처리
    → 상품 추천: 사용자 요구사항 분석 후 최적 상품 연결
    → 교육 중개: 전문 강의 및 교육 콘텐츠 연결
    → 수익 배분: 플랫폼 20% 수수료 / 챗봇 운영자 80% 자동 정산
    ↓
[5단계: 피상속] 사용자 사망 후에도 챗봇은 영속적으로 존속
    → 생전에 페르소나별·데이터별 피상속 허용/불허 사전 설정
    → 상속인을 사전 지정 (복수 가능, 접근 범위 차등 설정)
    → 사망 확인 시: 동의 항목 → 상속인에게 소유권 이전
                     비동의 항목 → 즉시 자동 완전 삭제 (복구 불가)
    → 피상속 후에도 상속인 하에서 챗봇이 계속 교육·성장·수익활동
    → ★ 디지털 유산 상속의 새로운 패러다임 ★
```

## 2-2. 기술 개발 현황 (TRL 6 — 시제품 완성 단계)

| 구분 | 개발 현황 | 완성도 |
|------|----------|--------|
| 웹 프론트엔드 (HTML/CSS/JS) | 챗봇 대화 인터페이스, 봇 생성 페이지, 대시보드 | 90% |
| 서버 API (Vercel Serverless) | 채팅 API, 봇 생성 API, 음성 인식/합성 API | 85% |
| 데이터베이스 (Supabase PostgreSQL) | 챗봇, 대화기록, 지식베이스, FAQ, 스킬 등 18개 테이블 (v2 크레딧·수익활동·Obsidian·스킬 연동 테이블 추가) | 90% |
| AI 멀티 모델 라우팅 | Gemini Flash → GPT-4o → Claude Sonnet → DeepSeek 폴백 체인 | 100% |
| 음성 입출력 (STT/TTS) | OpenAI Whisper + TTS-1 | 80% |
| 텔레그램 봇 연동 | 웹훅 기반 실시간 대화 + 음성 메시지 지원 | 100% |
| **특허 출원** | 멀티 페르소나 + 감성 AI 라우팅 + 영속적 라이프사이클(피상속 포함) | **출원 완료** |

### 기술 아키텍처

```
[사용자] ─── 웹/모바일/텔레그램 ───→ [Vercel CDN + Serverless]
                                            │
                    ┌───────────────────────┼───────────────────────┐
                    │                       │                       │
            [OpenRouter API]         [Supabase DB]           [OpenAI API]
          · Gemini Flash(1순위)     · PostgreSQL             · Whisper STT
          · GPT-4o(2순위)          · pgVector(RAG)          · TTS-1
          · Claude Sonnet(3순위)   · Auth + Storage
          · DeepSeek V3(4순위)     · 피상속 설정 관리
```

**원소스 멀티유즈**: 동일한 AI 모델 스택을 웹·텔레그램·카카오톡 등 모든 채널에서 공유하여 비용 최소화.

## 2-3. 경쟁력 확보 방안

### (1) 기술적 해자 (Moat)

| 해자 | 상세 |
|------|------|
| **특허 장벽** | 7대 핵심 구성요소(멀티 페르소나 이원 구조 + 감성 AI 라우팅 + 음성 5분 생성 + 교육·성장 체계 + 수익활동 중개 + 피상속 관리 + 배포) 특허 출원 완료 (청구항 22개) |

3. **특허 기반 기술 해자**: 핵심 기술을 특허로 보호하되, 특히 피상속 시스템은 MS·Meta 특허와도 근본적으로 차별화되는 독보적 기술
