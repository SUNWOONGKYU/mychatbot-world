# My Chatbot World — 요구사항 정의

> 작성일: 2026-03-31
> 소급 도입: 기존 구현 분석 + 사업계획서 기반

---

## 기능 요구사항 (Functional Requirements)

### FR-1: 챗봇 생성 (Create)
- FR-1.1: 음성 5분 인터뷰 → STT → AI 분석 → 자동 생성
- FR-1.2: 템플릿 기반 생성 (10개 직종 템플릿)
- FR-1.3: FAQ 자동 생성 (AI 분석 기반)
- FR-1.4: 멀티채널 배포 (URL, QR코드, 임베딩)

### FR-2: AI 대화 (Bot)
- FR-2.1: 멀티 페르소나 전환 (아바타형/도우미형)
- FR-2.2: 감성 슬라이더 (1~100) + 비용 슬라이더
- FR-2.3: 멀티 AI 자동 라우팅 (OpenRouter)
- FR-2.4: TTS/STT (음성 대화)
- FR-2.5: 대화 기록 저장·검색

### FR-3: 대시보드 (Home)
- FR-3.1: 지식 베이스 (KB) 관리·임베딩
- FR-3.2: 사용량 모니터링·크레딧 관리
- FR-3.3: 챗봇 설정 저장·클라우드 동기화

### FR-4: 학습 (School)
- FR-4.1: AI 시나리오 기반 학습 세션
- FR-4.2: 자동 채점·피드백
- FR-4.3: 멘토링 시스템

### FR-5: 스킬 마켓 (Skills)
- FR-5.1: 스킬 검색·설치·실행
- FR-5.2: 스킬 결제 (수수료 모델)
- FR-5.3: 리뷰·평점 시스템

### FR-6: 수익활동 (Jobs)
- FR-6.1: 채용 워크플로우
- FR-6.2: AI 매칭 알고리즘
- FR-6.3: 수수료 자동 정산

### FR-7: 커뮤니티 (Community)
- FR-7.1: 게시판 (CRUD + 좋아요·북마크)
- FR-7.2: 마당 (실시간 소통)
- FR-7.3: 카테고리별 분류

### FR-8: 수익 대시보드 (Business)
- FR-8.1: 수익 API (매출·정산 조회)
- FR-8.2: 결제수단 관리
- FR-8.3: 정산 리포트

### FR-9: MyPage / 피상속
- FR-9.1: 프로필 관리
- FR-9.2: 피상속인 지정·동의 관리
- FR-9.3: 디지털 유산 전환 프로세스

## 비기능 요구사항 (Non-Functional Requirements)

| 항목 | 요구사항 |
|------|---------|
| 성능 | 페이지 로드 < 3초, API 응답 < 500ms |
| 보안 | Supabase RLS, API 키 보호, XSS/CSRF 방지 |
| 가용성 | 99.5% (Vercel SLA) |
| 접근성 | 모바일 반응형, 한국어 우선 |
| 확장성 | 스킬 마켓 플러그인 구조, 멀티 AI 라우팅 |
