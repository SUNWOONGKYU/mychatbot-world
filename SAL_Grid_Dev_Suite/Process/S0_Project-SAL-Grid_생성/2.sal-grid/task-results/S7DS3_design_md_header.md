## 🎯 Design Principles (북극성)

MCW 디자인의 모든 결정은 다음 7개 원칙에 기반한다.
컴포넌트를 만들든, 토큰을 추가하든, 페이지를 리디자인하든 — 이 원칙이 판단 기준이다.

---

### 1. Clarity First — 명료함이 우선이다

> "인터페이스의 모든 요소는 단 하나의 의미를 전달해야 한다."

Accent 컬러(Indigo)는 버튼·링크·포커스에만. 색이 많아질수록 신호가 약해진다.
상태(성공/오류/경고/정보)는 항상 Semantic 컬러 4종으로만 전달한다.

---

### 2. Tokens Are Truth — 토큰이 사실이다

> "색상, 크기, 간격의 모든 결정은 토큰에서 시작하고 토큰에서 끝난다."

HEX, rgba, 원시 px 값을 코드에 직접 쓰지 않는다.
필요한 토큰이 없다면, 코드를 쓰기 전에 토큰을 먼저 추가한다.
토큰 한 곳을 바꾸면 시스템 전체에 전파된다 — 그것이 토큰의 목적이다.

---

### 3. Dark–Light Symmetry — 다크와 라이트는 대칭이다

> "모든 색상 결정은 다크와 라이트 모두에서 동등한 품질로 작동해야 한다."

새 시맨틱 토큰은 `.dark`와 `.light` 양쪽에 동시에 정의한다.
개발 후 반드시 라이트 모드로 전환하여 시각 확인한다.
`white`, `rgba(255,255,255,0.x)` 직접 사용은 라이트 모드 파손의 원인이다.

---

### 4. Motion Tells Direction — 모션은 방향을 말한다

> "움직임은 사용자에게 '무엇이 어디서 왔고 어디로 가는지'를 알려주는 안내자다."

드롭다운은 위에서 내려오고, Toast는 오른쪽에서 들어온다. 방향이 없는 모션은 장식이다.
5단 duration(75/150/250/350/500ms)과 4종 easing을 사용한다.
`prefers-reduced-motion` 분기는 선택이 아닌 필수다.

---

### 5. Accessible by Default — 접근성은 기본값이다

> "모든 사용자가 키보드만으로, 시각 보조 기기만으로 MCW를 완전히 사용할 수 있어야 한다."

`outline: none`은 쓰지 않는다. Focus ring은 2px Indigo + 2px offset.
텍스트와 배경의 대비율은 항상 WCAG AA(4.5:1) 이상이다.
ARIA 속성과 시맨틱 HTML은 접근성 검토 단계가 아닌 구현 단계에서 작성한다.

---

### 6. Korean First Citizen — 한국어가 일등 시민이다

> "한국어 텍스트는 영문과 동등한 가독성과 미학적 품질로 표시되어야 한다."

폰트는 Pretendard(한글) + Inter(영문) 이중 스택. 다른 폰트를 페이지별로 지정하지 않는다.
한국어 본문에는 `word-break: keep-all`을 적용한다.
한국어 line-height는 영문보다 0.1em 높게(body 1.5 → 1.6), 버튼 letter-spacing은 0.

---

### 7. Dense but Breathable — 밀도와 호흡의 균형

> "MCW는 정보가 많은 플랫폼이다. 밀도는 유지하되, 위계와 여백으로 숨을 쉬어야 한다."

모든 간격은 8pt 배수(4/8/12/16/20/24/32/48/64/96px). 예외 없이.
마케팅 1200px · 대시보드 1440px · 폼 480px — 콘텐츠 유형에 맞는 max-width를 쓴다.
Elevation 4단계(none/카드/드롭다운/모달)로 공간 위계를 명확히 한다.

---

### 원칙 우선순위 (충돌 시)

```
접근성(#5) > 토큰(#2) > 대칭(#3) > 명료함(#1) > 한국어(#6) > 밀도(#7) > 모션(#4)
```

접근성 위반은 즉시 수정. 나머지는 위 순서대로 우선 처리한다.
